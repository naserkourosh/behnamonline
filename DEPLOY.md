# راهنمای انتقال بهنام به سرور واقعی

## پیش‌نیازهای هاست
- **PHP 8.3** با اکستنشن‌های: `pdo_mysql` ، `curl` ، `gd` (کپچای ادمین) ، `mbstring` ، `fileinfo`
- **MySQL 8 یا MariaDB 10.4+** (یک دیتابیس + کاربر اختصاصی) — پروژه با هر دو سازگار است
- Apache با `mod_rewrite` (فایل `public/.htaccess` آماده است) — هاست اشتراکی cPanel/DirectAdmin کافی است
- **SSL/HTTPS** فعال (Let's Encrypt رایگان کافی است)

## چیزهایی که باید بخری / بگیری
| مورد | از کجا | کجا وارد می‌شود |
|---|---|---|
| مرچنت‌کد واقعی زرین‌پال | ثبت سایت در zarinpal.com با دامنهٔ واقعی | `.env` → `ZARINPAL_MERCHANT_ID` + `ZARINPAL_SANDBOX=false` + `PAYMENT_DRIVER=zarinpal` |
| پنل ملی‌پیامک + خط اختصاصی | melipayamak.com | پنل ادمین → پیامک‌ها → «اتصال به پنل» |
| الگوی (پترن) کد تایید | پنل ملی‌پیامک → ارسال متن خدماتی → ثبت و تایید متن | همان فرم، فیلد «کد پترن کد تایید» + کد پترن هر قالب |
| (اختیاری) وب‌سرویس پست ملی | شرکت ملی پست | `.env` → `POST_*` — تا آن موقع برآورد داخلی کار می‌کند |

## مراحل استقرار

### ۱) آپلود فایل‌ها
ریشهٔ سایت (docroot) باید به پوشهٔ **`public/`** اشاره کند تا `app/` و `.env` از وب قابل دسترس نباشند.
- اگر هاست اجازهٔ تغییر docroot می‌دهد: کل پروژه را آپلود کن و docroot را روی `public/` بگذار.
- اگر cPanel با `public_html` ثابت است: پروژه را در پوشه‌ای کنار `public_html` (مثلاً `~/behnam/`) بگذار و محتوای `public/` را داخل `public_html` کپی کن؛ سپس در `public_html/index.php` مسیر `require` را به `../behnam/` اصلاح کن.

### ۲) ساخت `.env` روی سرور (فایل `.env` هرگز آپلود/کامیت نمی‌شود)
از روی `.env.example` بساز و این‌ها را حتماً تنظیم کن:
```
APP_ENV=production
APP_DEBUG=false            ← خیلی مهم؛ نمایش خطاها خاموش شود
APP_URL=https://your-domain.ir
APP_KEY=<۳۲ کاراکتر تصادفی>   ← تولید: php -r "echo bin2hex(random_bytes(32));"
DB_*  = مشخصات دیتابیس هاست
SECURE_COOKIES=true        ← پشت HTTPS
PAYMENT_DRIVER=zarinpal
ZARINPAL_MERCHANT_ID=<مرچنت واقعی>
ZARINPAL_SANDBOX=false
SMS_DRIVER=melipayamak     ← یا بعداً از فرم پنل ادمین تنظیم کن (فرم بر .env اولویت دارد)
```

### ۳) دیتابیس
```
php database/migrate.php
```
- اگر می‌خواهی محصولات/تنظیماتی که در لوکال وارد کرده‌ای منتقل شود: از دیتابیس لوکال `mysqldump` بگیر و روی سرور import کن (پوشهٔ `public/uploads/` را هم کامل کپی کن — تصاویر آنجاست).
- `database/seed.php` را روی سرور **اجرا نکن** (دادهٔ نمایشی است).

### ۴) مجوز نوشتن
این پوشه‌ها باید برای PHP قابل نوشتن باشند:
```
storage/sessions  storage/logs  storage/cache  public/uploads
```

### ۵) امنیت — بلافاصله بعد از بالا آمدن سایت
1. **رمز ادمین پیش‌فرض (admin / admin1234) را عوض کن** — پنل ادمین → کاربران مدیریت.
2. در `public/.htaccess` خط HSTS را از کامنت درآور (فقط وقتی HTTPS پایدار است):
   `Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains"`
3. مطمئن شو `https://domain.ir/.env` خطای 403/404 می‌دهد (htaccess محافظت می‌کند؛ فقط تست کن).

### ۶) تنظیمات فروشگاه از پنل ادمین
- **پیامک‌ها → اتصال به پنل**: نام کاربری/رمز ملی‌پیامک + شماره خط + کد پترن کد تایید → ذخیره → دکمهٔ «بروزرسانی» اعتبار باید عدد واقعی نشان دهد (= اتصال سالم).
- **قالب‌های پیامک**: کد پترن هر قالب را (پس از تایید الگو در پنل ملی‌پیامک) وارد کن — ترتیب متغیرها زیر هر قالب نوشته شده.
- **ارسال و مناطق**: روش‌های ارسال (پست/پس‌کرایه/پیک گرگان) و آستانهٔ ارسال رایگان را نهایی کن.
- **تنظیمات**: اطلاعات تماس، اینماد و … .

## تست‌های پس از استقرار (چک‌لیست)
- [ ] ورود مشتری با OTP (پیامک واقعی برسد — از خط خدماتی/پترن)
- [ ] یک خرید واقعی کم‌مبلغ کامل: سبد → پرداخت زرین‌پال → پیامک پرداخت → دیدن سفارش در پنل ادمین → تایید/ارسال → پیامک کد رهگیری
- [ ] یک سفارش کارت‌به‌کارت + تایید دستی ادمین
- [ ] `https://domain.ir/sitemap.xml` و `robots.txt` باز شوند → ثبت در Google Search Console
- [ ] فرم ارسال گروهی پیامک با ۱–۲ شمارهٔ خودت
- [ ] گفتگوی آنلاین (ویجت مشتری ↔ پنل ادمین)
- [ ] برنامهٔ بکاپ منظم دیتابیس + پوشهٔ `public/uploads` (اکثر هاست‌ها JetBackup دارند)

## یادآوری‌های فنی
- ساعت اپ روی Asia/Tehran است و اتصال دیتابیس هم `time_zone` جلسه را خودش هم‌تراز می‌کند — تنظیم خاصی روی سرور لازم نیست.
- CSS از پیش build شده (`public/assets/css/app.css`) — روی سرور نیازی به Node/Tailwind نیست.
- لاگ خطاهای پیامک: `storage/logs/sms.log`.
